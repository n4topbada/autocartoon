# 카카오 알림톡 도입 준비

작성 기준: 2026-07-18 KST

이 문서는 국내 서비스인 워니바나나봇에 카카오 알림톡을 연결하기 위한 실행 기준이다. 카카오 로그인, 카카오페이, 카카오톡 채널, 알림톡은 서로 별도 서비스다. 현재 카카오 로그인이 동작하더라도 알림톡 발송 권한이 생기지는 않는다.

## 1. 조사 결론

- 알림톡은 카카오톡 채널 친구가 아닌 이용자에게도 보낼 수 있는 **정보성 메시지**다.
- 카카오가 공개한 단일 발송 API를 사업자가 바로 쓰는 방식이 아니라 카카오와 계약한 **공식 딜러사**를 통해 가입·발송한다.
- 사업자 카카오톡 채널, 비즈니스 채널 심사, 딜러사의 발신프로필 등록, 템플릿 심사를 모두 마쳐야 한다.
- 승인된 정형 템플릿만 보낼 수 있다. 광고 문구나 무관한 홍보를 섞으면 반려 또는 제재될 수 있다.
- 무료 기능이 아니다. 채널 개설과 비즈니스 채널 전환 자체와 별개로 실제 발송은 딜러사에서 건당 과금한다.
- 현재 코드에 솔라피 연동 골격이 있어 딜러사는 우선 SOLAPI로 정리한다. 다른 공식 딜러로 바꾸려면 어댑터만 교체한다.

## 2. 사업자가 먼저 할 일

1. 카카오비즈니스 통합회원으로 전환한다.
2. 워니바나나봇 명의의 카카오톡 채널을 만들고 공개 상태로 전환한다.
3. 사업자등록번호가 일치하는 **비즈니스 채널**로 전환 심사를 신청한다.
4. SOLAPI 사업자 계정을 만들고 채널을 발신프로필로 등록한다.
5. 문자 대체 발송을 사용할 경우 사업자 명의 발신번호도 등록한다.
6. 아래 템플릿을 등록하고 카카오 검수를 요청한다.
7. SOLAPI 잔액을 충전하고 운영 API Key와 API Secret을 발급한다.

비즈니스 채널 심사 서류는 신청자 권한에 따라 달라진다. 대표자는 카카오 인증서 본인 확인을 사용하고, 직원·대행 신청은 사업자등록증과 재직·위임 관계를 입증하는 서류가 추가될 수 있다. 제출 전 카카오의 최신 채널 만들기 안내를 다시 확인한다.

## 3. 권장 템플릿

### AI 생성 완료

```text
[워니바나나봇] 생성 완료

#{작업명} 생성이 완료되었습니다.
완료 시각: #{완료시각}

결과는 워니바나나봇 보관함에서 확인할 수 있습니다.
```

버튼: `결과 보기` / 웹 링크 / 해당 작업 또는 보관함 URL

### 생성 실패와 크레딧 반환

```text
[워니바나나봇] 작업 처리 안내

#{작업명} 생성에 실패했습니다.
사용한 #{크레딧}크레딧은 자동으로 반환했습니다.
처리 시각: #{처리시각}
```

### 지원 문의 접수

```text
[워니바나나봇] 새 지원 요청

사용자: #{사용자명}
문의 요약: #{메시지내용}
```

생성 완료와 실패 메시지는 이용자가 직접 시작한 작업의 상태 안내로 한정한다. 할인, 신규 상품, 친구 추가 유도 같은 광고성 문구는 섞지 않는다. 운영 공지처럼 비정형·간헐적 메시지는 알림톡이 아닌 인앱 공지 또는 별도 동의를 받은 광고 메시지로 처리한다.

## 4. 서비스에 추가로 필요한 기능

현재 User 모델에는 휴대전화번호가 없다. 사용자 완료 알림을 켜려면 다음 작업이 선행되어야 한다.

1. 휴대전화번호, 인증 시각, 알림톡 사용 여부를 저장하되 번호는 암호화하거나 별도 보호한다.
2. 설정 화면에 번호 인증, 완료 알림 켜기/끄기, 개인정보 수집 안내를 추가한다.
3. 카카오 로그인에서 전화번호가 항상 제공된다고 가정하지 않고 별도 번호 인증을 구현한다.
4. 작업이 `succeeded` 또는 `failed`로 처음 전환될 때만 발송하는 멱등 이벤트를 만든다.
5. 인앱 알림의 `notifiedAt`과 구분해 알림톡 요청·성공·실패·딜러 메시지 ID를 별도 기록한다.
6. 딜러사 결과 webhook을 검증하고 실패 재시도와 선택적 SMS 대체 발송을 관리한다.
7. 번호 삭제·변경, 알림 철회, 탈퇴와 보유기간을 개인정보처리방침에 반영한다.

## 5. 현재 코드 상태

- `src/lib/kakao-notify.ts`는 SOLAPI REST 호출 어댑터다.
- 기존 Bearer API Key 방식은 SOLAPI 규격과 달라 HMAC-SHA256 서명 방식으로 수정했다.
- 현재 호출부는 사용자 생성 완료가 아니라 **지원 문의를 운영자 번호로 알리는 용도**뿐이다.
- 환경변수와 승인 템플릿이 없으면 조용히 발송을 건너뛰며 이메일·DB 문의 접수는 계속된다.
- 사용자 완료 알림은 휴대전화번호와 수신 설정이 없으므로 아직 활성화하면 안 된다.

필요한 Secret Manager 항목:

```text
SOLAPI_API_KEY
SOLAPI_API_SECRET
SOLAPI_PFID
SOLAPI_SENDER_PHONE
SOLAPI_ADMIN_PHONE
SOLAPI_HELP_TEMPLATE_ID
```

## 6. 비용 판단

카카오는 딜러사마다 서비스 범위와 발송 단가가 다를 수 있다고 안내한다. SOLAPI 공개 직접 연동 단가는 현재 페이지 기준 알림톡 8원/건으로 표시되지만 부가세, 대체 문자, 부가 기능과 계약 조건은 결제 직전에 다시 확인해야 한다.

프로토타입에서는 다음처럼 시작하면 된다.

- 지원 문의 운영자 알림: 하루 최대 20건 제한
- 사용자 생성 완료: 이용자가 명시적으로 켠 경우에만 1작업 1건
- 여러 이미지가 한 작업에서 만들어지면 결과별이 아니라 작업 완료 1건
- 인앱 알림은 항상 유지하고 알림톡은 선택 기능으로 제공

- 사용자 번호 수집·인증·철회 UI

- 개인정보처리방침 최종 반영
