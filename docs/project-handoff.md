## 3. 확정된 제품 방향

1. 사용자가 AI API 키를 넣지 않는다. 플랫폼 소유 GCP Vertex AI가 Gemini와 Veo를 제공한다.
2. 신규 사용자에게 30크레딧을 한 번 지급하고, 외부 비용이 드는 AI·OCR·TTS·영상 호출과 유료 캐릭터 구매에 크레딧을 차감한다.
3. 실패한 AI 작업은 같은 원장 참조 키로 자동 환불한다.
4. 충전은 카카오페이 단건 결제를 사용한다. 구독보다 선불 크레딧을 우선한다.
5. 레퍼런스의 기능 흐름은 참고하지만 브랜드, 문구, 이미지, 소스 코드와 보안 결함은 복제하지 않는다.
6. 생성 작업은 페이지 생명주기와 분리된 영속 작업으로 만들고 재접속 복구와 완료 알림을 제공한다.

## 4. 현재 구현된 기능

### 4.1 인증과 계정

- 이메일 회원가입, 이메일 확인, 로그인, 로그아웃
- 비밀번호 분실 시 12자 영문·숫자 임시 비밀번호 이메일 발급
- 임시 비밀번호 로그인 후 강제 변경, 설정 내 비밀번호 변경
- 사용자 세션 목록과 개별 세션 관리
- HttpOnly `iron-session` 쿠키, 관리자 역할 검사
- 카카오 OAuth 시작·콜백, CSRF `state` 검증, 검증 이메일 기준 기존 계정 연결
- 계정 삭제 시 카카오 ID 제거와 개인정보 비식별화

### 4.2 캐릭터와 자산

- 캐릭터 그룹, 프리셋, 구매, 대표 이미지, 방향별 이미지 관리
- 정면·좌측·우측·후면 참조와 대표 이미지 선택
- 생성 결과에서 캐릭터 등록
- 최대 4명 캐릭터를 한 장면에 배치
- 사용자별 캐릭터, 관리자 전체 사용자 전환 보기
- 원본과 512px WebP 썸네일 분리 저장
- 아카이브, 즐겨찾기, 태그, 다운로드, 삭제

### 4.3 이미지·배경·제스처 생성

- 텍스트, 스케치, 이미지 편집, 이미지 변환 생성 모드
- 1K·2K, 출력 수, 비율, 사용자 참조 이미지
- 캐릭터 방향과 장면 방향을 반영한 프롬프트
- 단일 및 다중 캐릭터 제스처/장면 생성
- 저밀도 배경 프롬프트 가드레일
- 원본 사진과 스타일 참조를 사용하는 배경 생성
- 생성 입력 크기·MIME·프롬프트 검증

### 4.4 영속 작업과 비용 처리

- `GenerationJob`, `GenerationArtifact`, `CreditLedger`
- Vercel Workflow 기반 이미지·Veo 장기 작업
- queued/processing/storing/completed/failed 단계와 실제 진행 상태
- 페이지 재접속 후 작업 복구, 완료 알림, 히스토리 자동 갱신
- 작업 재시도, idempotency, 중복 차감 방지
- 모델·저장 실패 시 원자적 자동 환불
- Blob 부분 업로드 실패 시 보상 삭제

### 4.5 제작 스튜디오

- `CreativeProject`, `ProjectCut`, `ProjectAsset`, 저장 기획서
- 프로젝트·컷 CRUD, 컷 순서, 대사, 장면 지시, 길이
- 이미지·영상 직접 업로드와 공유 자산 참조
- AI 기획안에서 프로젝트와 컷 생성
- OCR, 영상 대사 플랜 생성
- 기존 CanvasEditor를 스튜디오 컷에 연결
- 레이어, 이미지, 텍스트, 말풍선, 크롭, 배경 제거, 효과음·캡션 프리셋

### 4.6 영상과 음성

- Veo 3.1 Fast 텍스트·이미지 기반 영상
- 9:16·16:9, 4·6·8초, 720p·1080p, 오디오 옵션
- 영상 작업 진행 상태와 결과 자산 저장
- Google Cloud TTS 음성 미리듣기
- 숏폼 컷 업로드와 브라우저 편집 화면

### 4.7 캐릭터 디렉터와 운영

- 관리자와 허용 계정용 캐릭터 설계 메뉴
- 대화로 이름, 외모, 신상, 성격, 말투, 지식, 기타 항목 생성
- 동적 항목 추가, 항목별 복사, 시스템 프롬프트 보기
- 관리자 사용자 목록, 역할·크레딧·카카오 연결·결제 건수 확인
- 관리자 수동 크레딧 지급과 원장 기록
- 지원 요청 이메일, 도움말 AI 채팅, 관리자 지식 관리

### 4.8 커뮤니티와 외부 연동

- 게시글, 댓글, 좋아요, 고정, 삭제 권한
- 콘텐츠 슬롯과 프롬프트 프리셋
- Instagram OAuth·게시·인사이트 API 코드
- Instagram 메뉴는 제품 UI에서 현재 비활성

### 4.9 크레딧과 결제

- 웰컴 30크레딧, 기존 일반 사용자 일괄 1회 지급
- 서버 소유 작업별 요율과 충전 상품
- 지갑 잔액, 상품, 작업별 비용, 충전·차감·환불 내역
- 카카오페이 ready/approve/cancel/fail 라우트
- 상품·금액·주문자·TID 서버 검증
- 중복 콜백 방지와 승인 후 적립의 원자성
- 승인 응답 유실 시 카카오페이 주문 조회로 자동 복구

## 5. 기술 구조와 주요 데이터

```text
Next.js UI
  -> Route Handler + iron-session
  -> PostgreSQL / Prisma
  -> GenerationJob / CreditPayment / CreditLedger
  -> Vercel Workflow
  -> Vertex AI Gemini / Veo / Google Cloud TTS
  -> Vercel Blob 원본 + 썸네일
  -> ProjectAsset / ProjectCut / GenerationArtifact
```

주요 모델:

- 사용자·인증: `User`, `UserSession`
- 캐릭터: `CharacterGroup`, `CharacterPreset`, `PresetImage`, `PurchasedPreset`
- 생성: `GenerationRequest`, `GenerationJob`, `GenerationArtifact`, `GeneratedImage`
- 결제: `CreditLedger`, `CreditPayment`
- 제작: `CreativeProject`, `SavedProjectBrief`, `ProjectCut`, `ProjectAsset`
- 커뮤니티: `BoardPost`, `BoardComment`, `BoardLike`
- 콘텐츠·연동: `Content`, `ContentSlot`, `PromptPreset`, `InstagramAccount`, `InstagramPost`

## 6. 레퍼런스 대비 기능 차이

상태 의미:

- `우위`: 현재 AutoCartoon이 기능 또는 보안 구조상 더 낫다.
- `동등`: 핵심 사용 흐름을 제공한다.
- `부분`: 기반은 있으나 레퍼런스의 완성도나 세부 기능이 부족하다.
- `열위`: 레퍼런스에는 있고 현재 제품에는 사실상 없다.
- `방향 다름`: 의도적으로 다른 사업·기술 모델을 선택했다.

| 캐릭터 디렉터 | 확인되지 않음 | 대화형 페르소나 설계기 | 우위 | 관리자 전용 독자 기능 |

## 8. 현재 잔여 위험과 알려진 제한

1. 카카오 로그인과 카카오페이는 실제 키가 없어 운영 E2E를 아직 하지 못했다.
2. 카카오페이 승인 응답 유실은 주문 조회로 복구하지만, 실제 가맹점 환경의 오류 코드와 정산 파일 대사는 검증 전이다.
3. 배경 3단계 작업 일부가 브라우저 상태에 남아 새로고침 복구 수준이 이미지·영상 job보다 낮다.
4. 아카이브, 저장 배경, 프로젝트 자산이 서로 다른 화면·API라 통합 검색 경험이 부족하다.
5. TTS는 미리듣기 수준이며 캐릭터 보이스 자산과 최종 영상 합성 계약이 없다.
6. 캔버스는 기능이 많아졌지만 Fabric.js 계열 객체 편집기와 비교하면 문서 모델·자동 저장·고급 조작이 부족하다.
7. 플랫폼 과금표는 기술적으로 적용됐지만 실제 원가·세금·결제 수수료를 반영한 사업 검증 전이다.

## 10. 후속 에이전트 작업 원칙

1. 기능이 화면에 있다는 이유만으로 구현 완료로 표시하지 않는다. API, DB, 실패·재시도, 권한, 모바일까지 확인한다.
2. 레퍼런스의 문구·디자인·자산을 그대로 복사하지 않는다. 사용자 흐름과 동작 계약만 독자 구현한다.
3. 결제와 크레딧 변경은 항상 서버 소유 금액, idempotency, 원장, 재조정 경로를 갖춘다.
4. 외부 AI 호출은 성공 결과가 영속 저장된 경우에만 최종 과금하고 실패 시 반대 분개한다.
5. 비밀번호와 API 키는 추적 문서나 Git에 기록하지 않는다.
6. 구현 후 이 문서의 기능표, 잔여 위험, 검증 기록을 함께 갱신한다.
