# Toon Agent 기능 역설계 조사

조사일: 2026-07-16
대상: `https://app.toonagent.co.kr`
목표: AutoCartoon이 레퍼런스의 사용자 워크플로를 기능적으로 지원하기 위한 제품·기술 기준선 수립

구현 업데이트: 사용자 BYOK는 도입하지 않고 플랫폼 소유 Vertex AI를 사용하기로 확정했다. 조사 뒤 GenerationJob, CreditLedger, Vercel Workflow, 프로젝트·컷·자산, 4면 참조, 제스처, 캔버스 연결, Veo 영상 생성까지 AutoCartoon에 반영했다. 아래 8~11절의 "현재"와 우선순위는 조사 시점 기준이며, 최신 구현 상태는 13절에 정리한다.

> 이 문서는 기능과 동작을 분석한 자료다. 레퍼런스의 소스 코드, 문구, 이미지, 브랜드 자산을 복제하지 않는다. 실제 구현은 AutoCartoon의 디자인과 데이터 모델에 맞춰 독자적으로 작성한다.

## 1. 조사 범위와 신뢰도

확인 방법은 다음 세 가지로 분리했다.

- **화면 확인**: 로그인과 인증 계정의 홈, 생성기, 자산 관리, 프로젝트, 캔버스, 설정, 툰 광장, 영상 제작 화면까지 확인했다. 결제·생성·게시·삭제의 최종 실행은 하지 않았다.
- **공개 네트워크/번들 확인**: 브라우저에 공개 배포된 JavaScript 번들, 공개 API 응답, 응답 헤더에서 화면 구성과 요청 계약을 확인했다.
- **로컬 소스 확인**: 현재 AutoCartoon 저장소의 컴포넌트, API, Prisma 모델, 인증·생성·저장 흐름을 확인하고 프로덕션 빌드를 실행했다.

제공 계정으로 로그인해 실제 권한과 초기 상태를 검증했다. 로그인 전에는 기기 2대 제한이 걸렸고, 사용자가 종료한 Windows 세션을 팝업에서 명시적으로 철회한 뒤 정상 로그인했다. 일반 로그아웃이 브라우저 JWT만 삭제하고 서버 기기 세션은 철회하지 않는 동작도 정적 번들과 실제 화면에서 함께 확인했다.

## 2. 핵심 결론

1. Toon Agent의 중심 제품은 단일 이미지 생성기가 아니라 `캐릭터 자산 -> 제스처/배경 자산 -> 프로젝트/컷 -> 캔버스 편집 -> 내보내기`로 이어지는 제작 워크스페이스다.
2. 체감 UX의 핵심은 비동기 생성 작업의 영속화다. 생성 중 페이지를 벗어나도 작업을 복구하고, 완료 결과를 히스토리에 자동 반영한다.
3. AutoCartoon은 캐릭터 프리셋, 다중 캐릭터, 썸네일, 배경 생성, 간단 캔버스, 콘텐츠·게시판, 관리자용 캐릭터 설계 기능을 이미 보유한다. 반면 프로젝트/컷, durable job, 4면 캐릭터, 보이스, BYOK, 고급 캔버스는 데이터 모델부터 새로 필요하다.
4. 레퍼런스 번들에는 Toon Agent 개인 기능 외에 Plurank 관리자·마케팅·CX 기능이 대량 포함돼 있다. 이것을 모두 1차 범위로 잡으면 제품이 불필요하게 비대해진다. 개인용 제작 기능과 숨은 플랫폼 기능을 별도 범위로 확정해야 한다.
5. 레퍼런스의 보안·제품 결함까지 그대로 따라가면 안 된다. JWT의 `localStorage` 저장, 일반 로그아웃 시 기기 세션 미철회, 거대한 단일 번들, 브랜드/약관 불일치 등은 개선 대상으로 본다.

## 3. 레퍼런스 제품 구조

### 3.1 사용자 진입

- 이메일/비밀번호 로그인
- 카카오 로그인
- 이메일 회원가입 전 휴대폰 본인인증
- 비밀번호 찾기: 이메일 입력 -> 휴대폰 본인인증 -> 새 비밀번호 설정
- 이용약관, 개인정보처리방침, 환불정책
- 한국어/영어/일본어 다국어 리소스

관찰된 일관성 문제:

- Toon Agent 화면의 약관·개인정보 문서가 Plurank 공용 문서와 섞여 있다.
- 회원가입의 기업회원 링크가 `/start`로 이동하지만 실제 화면은 Plurank 브랜드 분석 온보딩이다.
- 비밀번호 최소 길이가 가입/초기화 화면과 계정 설정 화면에서 서로 다르다.

### 3.2 개인 계정 메뉴

제공된 전문가 계정에서 다음 메뉴가 실제로 노출됐다.

- 홈/온보딩
- 툰 생성기
- 캐릭터 생성기
- 제스처 생성기
- 배경 생성기
- 내 캐릭터
- 내 생성물
- 툰 광장
- 영상 만들기
- 설정
  - AI API 키
  - 구독/결제
  - 계정
  - 보안/기기

홈은 `캐릭터 생성 -> 제스처 생성 -> 툰 생성` 3단계 온보딩, 보유 캐릭터/웹툰 수, 공지, 빠른 실행 카드를 제공한다. 제공 계정에는 캐릭터 3개와 웹툰 9편이 표시됐고, Gemini/OpenAI/Remove.bg 키가 등록된 상태였다. 키 값은 마스킹돼 있었으며 조사 자료에는 저장하지 않았다.

## 4. 사용자 기능 상세

### 4.1 캐릭터 생성기

입력 항목:

- 성별: 여성, 남성, 중성적, 소년, 소녀
- 연령대: 어린이, 청소년, 20대, 30대, 중년, 노년
- 분위기/스타일: 귀여운, 시크한, 강렬한, 신비로운, 발랄한, 차분한, 판타지
- 헤어: 긴 생머리, 단발, 포니테일, 곱슬, 짧은 머리, 금발, 흑발 등
- 의상: 교복, 캐주얼, 정장, 후드티, 한복, 판타지 갑옷, 파일럿 슈트 등
- 자유 프롬프트 최대 2,000자
- 흰 배경 또는 장면 배경

모델 선택:

- Gemini 계열: 빠르고 저렴하며 동일 캐릭터 유지에 강점이 있다는 안내
- Gemini Pro 이미지 모델
- OpenAI 이미지 모델과 low/medium/high 품질
- 화면에 품질별 예상 원화 비용을 표시

동작:

- 1:1 이미지 한 장을 생성하고 결과/히스토리를 표시한다.
- 다운로드, 동일 설정 재생성, 내 캐릭터 저장을 지원한다.
- 생성 결과를 내 캐릭터로 저장할 때 한 장을 네 방향 이미지에 복제하는 흐름이 있어, 실제 4면도 생성은 아니다.

주요 요청 계약:

- `POST /api/generatewebtoon/characters/generate`
- prompt, provider, tier, OpenAI quality, background 여부, 1:1, 이미지 1장, 저장 여부를 전달한다.

### 4.2 내 캐릭터

- 이름과 선택 설명
- 정면, 좌측, 우측, 후면 4장 업로드
- 장당 최대 10MB
- 기본 캐릭터 지정
- 추가, 수정, 삭제
- 직접 업로드와 AI 생성 두 진입점

보이스:

- 캐릭터당 최대 3개
- 첫 보이스를 영상용 기본값으로 사용
- 의미 기반 보이스 검색
- 테스트 문장과 미리듣기
- 순서 변경, 기본값 지정, 제거

보이스 공급자는 공개 번들만으로 확정할 수 없다.

### 4.3 제스처 생성기

단일 캐릭터 모드:

- 저장 캐릭터 선택, 기본 캐릭터 자동 선택
- 선택 스타일 이미지
- 다중 참조 이미지
- 포즈/상황 프롬프트
- 배경 포함/제외
- 카메라 앵글
- 모델/품질 선택
- 이전 결과를 편집 소스, 캐릭터, 스타일, 참조로 다시 사용

2인 장면 모드:

- 캐릭터 A/B를 각각 선택
- 캐릭터별 동작 지시
- 선택 장면 지시
- 참조 이미지 최대 3장
- 배경, 앵글, 모델/품질
- 말풍선·대사는 생성하지 않는다는 안내

카메라 앵글:

- 정면, 3/4, 측면, 로우, 하이, 버드아이, 오버숄더, 더치, 클로즈업, 익스트림 클로즈업

비동기 작업 UX:

- `POST /gesture/jobs`로 작업 생성
- `/gesture/jobs`를 약 3.5초 간격으로 조회
- queued/processing 작업을 페이지 진입 시 복구
- 캐릭터 분석, 포즈, 디테일, 마무리 단계 문구
- 15분 타임아웃과 연속 조회 실패 시 히스토리 확인 안내
- 재시도 쿨다운
- 커서 기반 과거 히스토리와 무한 스크롤

이 흐름은 AutoCartoon에서 가장 먼저 가져올 가치가 높다.

### 4.4 배경 생성기

- 원본 사진 선택
- 스타일 이미지 선택
- 텍스트 프롬프트/추가 요청
- 카메라 앵글
- 모델/품질
- 원본 사진이 있으면 스타일 이미지 필수
- 원본 사진이 없으면 텍스트 프롬프트 필수
- 결과 다운로드와 배경 히스토리

요청은 multipart 기반 `/background/transform`, 히스토리는 `type=background` 필터를 사용한다.

### 4.5 내 생성물

- 전체, 제스처, 캐릭터, 배경, 누끼 필터
- 페이지당 24개
- 썸네일 그리드
- 프롬프트와 생성일
- 원본 열기/다운로드/삭제
- 삭제 시 확보한 저장 용량 표시
- 원본과 별도 썸네일 URL 사용

제공 계정에서는 총 295개가 표시됐고 24개씩 13페이지로 나뉘었다. 제스처 페이지의 최근 결과 패널과 내 생성물 보관함은 같은 생성 히스토리를 다른 문맥으로 노출한다.

### 4.6 프로젝트

- 개인 Gemini/OpenAI API 키가 없으면 설정으로 이동
- 빈 프로젝트 또는 마크다운 기획서로 시작
- 프로젝트 이름 변경/삭제
- 카드에 대표 썸네일, 컷 수, 수정일

제공 계정에는 프로젝트 9개가 있었다. 프로젝트 목록 카드의 `0컷` 표기와 실제 편집기에서 불러온 4개 페이지가 일치하지 않는 사례를 확인했다. 영상 소스 선택 화면은 같은 프로젝트를 `사용 가능한 컷 1개`로 계산했다. 프로젝트 카드, 편집 문서, 영상용 렌더 자산이 서로 다른 카운트를 사용하고 있어 데이터 기준을 하나로 통합할 필요가 있다.

마크다운 기획서 항목:

- 개요, 메시지, USP, CTA
- Hook, Relate, Twist, Punchline, CTA
- 컷별 대사, 텍스트, 장면 지시
- 주 캐릭터와 모델/품질 선택
- 기획서 분석과 이미지 일괄 생성

고급 플랫폼 계정용 흐름에는 PDF, 이미지, DOCX, ZIP, TXT, 블로그 가져오기, 캐릭터 별칭/역할, 스타일 참조, 의료광고 준수, AI 텍스트 삽입, 취소 가능한 비동기 진행 상태가 포함된다. 개인 계정 범위와 분리해야 한다.

### 4.7 캔버스 편집기

기본 문서 구조:

- 프로젝트 아래 여러 컷/페이지
- 컷 추가, 이름 변경, 순서 변경, 복제, 삭제
- 표지 컷 지정
- 자동 저장과 저장 상태 표시
- 현재 컷 PNG, 전체 컷 ZIP 내보내기
- 이미지/URL 드래그앤드롭

캔버스 규격:

- 1080x1080
- 1080x1350
- 960x1280
- 800x1100 레거시

편집 기능:

- 선택, 이동, 크기 조절, 회전
- 텍스트, 말풍선, 도형, 펜, 압력 드로잉, 지우개, 크롭
- 이미지, 캐릭터, 제스처, 배경, 효과음 자산
- 폰트, 크기, 색상, 외곽선, 정렬, 굵기, 기울임, 밑줄, 행간, 자간
- 말풍선 스타일, 채우기/투명도, 꼬리 핸들
- 사각형, 원, 타원, 선, 화살표와 테두리/채우기/그라데이션
- 레이어 순서, 잠금, 그룹/해제, 정렬/분배, 가이드
- 투명 배경, 크롭 마스크, 뒤집기, 불투명도
- 단색/그라데이션/텍스처 배경
- 워터마크와 캡션 스타일 일괄 적용

AI 편집:

- 전체/가이드/정밀 영역 재생성
- AI 영역 감지 또는 직접 영역 그리기
- 다른 프로젝트 컷을 정체성/스타일 참조로 사용
- 캐릭터 디자인 변형 허용 옵션
- 전후 비교, 적용, 재시도, 취소
- 텍스트 추출 후 지우기/인페인트
- 선택 영역 캐릭터 교체
- 배경 제거
- 스토리보드 패널에서 구도, 전경/배경, 포즈, 캡션, 대사 삽입

레퍼런스는 Fabric.js 계열 캔버스를 사용한다. AutoCartoon의 현재 직접 Canvas 2D 구현을 이 수준까지 확장하는 것보다 검증된 객체 캔버스 라이브러리로 문서 모델을 새로 설계하는 편이 현실적이다.

조회처럼 보이는 동작의 쓰기 부작용도 확인했다. 기존 프로젝트를 열고 `AI로 컷 다시 그리기` 대화상자만 연 뒤 생성하지 않고 취소했지만, 기존 업로드 이미지를 컷으로 등록한다는 상태가 표시되고 프로젝트 수정 시각이 현재 시각으로 바뀌었다. 편집기 진입·마이그레이션·자동 저장을 분리하고, 사용자 변경이 없으면 `updatedAt`을 갱신하지 않는 것이 바람직하다.

### 4.8 영상 만들기

- 이미지 직접 업로드 또는 내 인스타툰 프로젝트 선택
- 컷 이미지 최대 30장

## 10. 현재 코드에서 발견한 위험

### P0: 생성 요청 복구 불가

`POST /api/generate`가 외부 AI 완료까지 기다리는 동기 요청이다. 클라이언트는 120초 후 요청을 중단하지만 서버 작업이 계속될 수 있고, 사용자는 성공 여부를 알 수 없다. 새로고침하면 진행 상태도 사라진다.

개선:

- 먼저 GenerationJob을 DB에 `queued`로 생성
- idempotency key로 중복 요청 방지
- worker/workflow에서 생성 실행
- `queued -> processing -> storing -> completed/failed` 상태 저장
- UI는 SSE 또는 폴링으로 복구
- 성공 결과는 히스토리에 즉시 연결

### P0: 부분 저장과 환불 정합성

AI 결과 뒤 원본 Blob 업로드와 DB 이미지 생성이 `Promise.all`로 진행된다. 여러 이미지 중 일부만 저장되고 나머지가 실패하면 부분 DB 레코드와 고아 Blob이 남을 수 있고, 요청 전체 크레딧은 환불될 수 있다.

개선:

- 작업·과금 ledger를 한 ID로 연결
- Blob 업로드 성공 목록을 추적해 실패 시 보상 삭제
- DB 저장은 트랜잭션 또는 명시적 최종화 단계 사용
- 환불을 사용량 단순 감소가 아닌 ledger 반대 분개로 처리

### P1: 진행률이 실제 서버 상태가 아님

현재 `요청 준비 -> 장면 그리기 -> 품질 다듬기`는 클라이언트 경과 시간으로 바뀌는 문구다. 사용성에는 도움이 되지만 실제 서버 단계나 실패/재시도를 반영하지 못한다.

### P1: 히스토리 확장성

현재 히스토리는 최대 20개 기본 조회이고 cursor/page UI가 없다. 이미지가 쌓이면 오래된 결과 접근이 어렵다.

### P1: 배경 작업 유실

배경 생성의 3단계 결과와 선택 상태가 React 로컬 상태에만 있어 새로고침/탭 이동 시 복구되지 않는다.

### P1: 품질 게이트 부재

- `npm run build`: 성공
- `npm run lint`: ESLint 설정 선택 프롬프트가 떠서 비대화형 CI에서 실패
- 자동 테스트 스크립트 없음
- `ARCHITECTURE.md`의 모델 수, API 수, 일부 기능 설명이 현재 코드보다 오래됨

인증된 주요 화면을 순회하는 동안 브라우저 오류·경고 로그는 없었다. 유료 생성, 구독 변경, API 키 저장, 게시, 삭제는 조사 중 실행하지 않았다. 기존 프로젝트 편집기 진입 과정에서 레퍼런스 자체의 자동 정규화/저장으로 수정 시각이 한 차례 갱신됐지만 AI 생성이나 크레딧 차감은 실행되지 않았다.

## 13. 확정된 방향과 최신 구현 상태

확정된 제품 결정:

1. 사용자 API 키 입력 없이 플랫폼 소유 Vertex AI 프로젝트가 Gemini와 Veo를 제공한다.
2. 레퍼런스의 브랜드, 문구, 이미지, 소스 코드를 복제하지 않고 기능과 제작 흐름을 독자 구현한다.
3. 개인용 제작 기능을 먼저 완성하되 임의의 1차 범위로 잘라 두지 않고 현재 저장소에서 안전하게 구현 가능한 범위를 계속 넓힌다.
4. AI 비용은 내부 크레딧과 작업 원장으로 통제한다.
5. Vercel은 OIDC Workload Identity Federation으로 Google Cloud에 접근하며 장기 서비스 계정 키를 배포하지 않는다.

이번 구현에서 완료한 항목:

- `GenerationJob`, `GenerationArtifact`, `CreditLedger`, idempotency key
- Vercel Workflow 기반 이미지 및 Veo 장기 작업
- 실제 서버 단계 진행률, 재접속 복구, 완료 알림, 실패 재시도와 자동 환불
- `CreativeProject`, `ProjectCut`, `ProjectAsset` 및 통합 스튜디오
- 프로젝트/컷 CRUD, 컷 대사와 길이, 직접 자산 업로드
- 정면·좌측·우측·후면 캐릭터 이미지 분류와 단일 캐릭터 생성 시 4면 참조 사용
- 장면 및 제스처 생성, 최대 4명 캐릭터 선택
- 기존 레이어·크롭·배경 제거·말풍선 캔버스를 스튜디오 컷에 연결
- 고해상도 캔버스의 브라우저 직접 Blob 업로드, 공유 자산 참조 보호
- Veo 3.1 Fast의 텍스트/이미지 기반 영상, 9:16·16:9, 4·6·8초, 720p·1080p, 오디오 옵션
- 실패 상태·크레딧 환불 원장의 원자적 처리와 Blob 부분 업로드 보상 삭제
- Vertex AI 전용 위치 분리: Gemini `global`, Veo `us-central1`
- Vercel OIDC와 GCP Workload Identity Pool, 최소 권한 서비스 계정
- ESLint 비대화형 구성, 단위 테스트, TypeScript 검사, 프로덕션 빌드

아직 레퍼런스만으로 확정하지 못한 항목:

- 캐릭터 보이스의 실제 공급자, 음성 복제 방식, 상업 이용 계약
- 최종 영상에서 외부 TTS 내레이션과 컷 영상을 합성하는 서버 렌더러 계약
- 레퍼런스의 고급 플랫폼 계정에만 보이는 마케팅/CX 모듈의 실제 권한별 동작
- 결제, 게시, 신고, 예약 게시를 최종 제출했을 때의 비공개 서버 규칙

현재 AutoCartoon은 보이스 영역을 브라우저 음성 미리듣기와 Veo 자체 오디오로 제공한다. 서버 TTS, 음성 복제, 내레이션 합성은 공급자와 라이선스가 확정되기 전 임의 구현하지 않는다.
