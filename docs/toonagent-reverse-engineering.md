# Toon Agent 기능 역설계와 TO-BE 기준

최종 갱신: 2026-07-18 KST

레퍼런스: `https://app.toonagent.co.kr`

목표: WONY AutoCartoon이 레퍼런스의 개인 창작 워크플로를 기능적으로 동등 이상으로 제공한다.

> 이 문서는 화면과 공개 클라이언트 동작을 기준으로 기능·요청 흐름을 분석한 자료다. 레퍼런스의 소스 코드, 문구, 이미지, 브랜드 자산은 복제하지 않는다. 접속 비밀은 Git에서 제외한 `docs/access-credentials.private.md`에만 둔다.

## 1. 조사 방법과 신뢰도

- 인증 계정으로 홈, 생성기, 자산, 프로젝트, 캔버스, 영상, 커뮤니티, 설정을 순회했다.
- 화면의 입력·상태·오류·폴링 동작과 브라우저에 공개된 요청 계약을 함께 확인했다.
- 결제, 유료 생성, 게시, 삭제, API 키 변경처럼 비용이나 데이터 변경이 생기는 최종 제출은 실행하지 않았다.
- 2026-07-17 인증 세션으로 프로젝트와 고급 캔버스를 다시 열어 각 도구·패널·컷 다시 그리기 메뉴를 재검증했다.
- 2026-07-18 인증 세션으로 홈, 캐릭터, 제스처 1인·2인, 배경, 보관함, 프로젝트 기획서, 캔버스, 광장, 영상, 설정을 다시 순회했다.
- 모델 공급자 이름이나 내부 기술 스택이 아니라 사용자가 같은 목적을 끝낼 수 있는지를 기준으로 다시 판정했다.
- AutoCartoon은 UI 존재 여부만 보지 않고 Route Handler, Prisma 소유권, 실패 처리, 크레딧 원장, 프로덕션 빌드까지 확인했다.

상태 정의:

- **우위**: 핵심 흐름을 포함하면서 범위·보안·복구 중 하나 이상이 더 낫다.
- **동등**: 사용자가 같은 목적을 끝까지 달성할 수 있다.
- **부분**: 핵심 기반은 있으나 세부 도구나 운영 완성도가 부족하다.
- **미구현**: 대응 흐름이 없다.
- **외부 결정**: 계약, 사업자 정보, 공급자 계정 또는 정책 확정 없이는 완료하면 안 된다.
- **방향 다름**: 의도적으로 다른 사업·기술 모델을 선택했다.

## 2. 레퍼런스 기능 역설계

### 2.1 진입, 계정, 보안

- 이메일/비밀번호, 카카오 로그인
- 회원가입 전 휴대폰 본인확인
- 비밀번호 찾기 시 휴대폰 본인확인 후 새 비밀번호 설정
- 한국어, 영어, 일본어
- 이용약관, 개인정보처리방침, 환불정책
- 기기 최대 2대, 세션 목록과 개별/일괄 철회
- 일반 로그아웃은 브라우저 JWT만 제거하고 서버 기기 세션은 남기는 결함이 관찰됨
- JWT를 `localStorage`에 보관하므로 XSS 토큰 탈취 면에서 HttpOnly 쿠키보다 불리함

### 2.2 홈과 온보딩

- 보유 캐릭터·웹툰 수, 공지, 최근 상태
- `캐릭터 생성 -> 제스처 생성 -> 툰 생성` 3단계 온보딩
- 각 제작 도구로 이동하는 빠른 실행

### 2.3 캐릭터 생성기

- 성별, 연령대, 분위기, 헤어, 의상, 그림 스타일, 배경 여부의 구조화 입력
- 자유 프롬프트 최대 2,000자
- Gemini, Gemini Pro 이미지, OpenAI 이미지와 품질 선택
- 모델·품질별 예상 비용 표시
- 1:1 이미지 생성, 진행 상태, 히스토리, 다운로드, 재생성, 내 캐릭터 저장
- 생성 결과 한 장을 네 방향 슬롯에 복제하는 저장 흐름이 있어 실제 4면도 자동 생성은 아님

### 2.4 내 캐릭터와 보이스

- 이름, 설명, 정면·좌·우·후면 이미지, 장당 최대 10MB
- 기본 캐릭터 지정, 추가·수정·삭제
- 캐릭터당 보이스 최대 3개, 첫 보이스를 영상 기본값으로 사용
- 의미 기반 보이스 검색, 테스트 문장, 미리듣기, 순서 변경, 제거

### 2.5 제스처 생성

- 단일 캐릭터: 캐릭터·스타일·다중 참조·포즈·배경·앵글·모델·품질
- 2인 장면: 캐릭터 A/B, 캐릭터별 행동, 장면 지시, 참조 최대 3장
- 정면, 3/4, 측면, 로우, 하이, 버드아이, 오버숄더, 더치, 클로즈업 등 앵글
- 이전 결과를 편집 소스, 캐릭터, 스타일, 참조로 재사용
- 작업 생성 후 약 3.5초 폴링, queued/processing 복구, 단계 문구, 15분 제한, 재시도 쿨다운
- 커서 기반 히스토리와 무한 스크롤

### 2.6 배경 생성

- 원본 사진, 스타일 이미지, 텍스트 추가 요청, 카메라 앵글, 모델·품질
- 원본이 있으면 스타일 이미지가 필요하고, 원본이 없으면 텍스트가 필요함
- 결과 다운로드와 전용 히스토리

### 2.7 내 생성물

- 전체, 제스처, 캐릭터, 배경, 누끼 필터
- 페이지당 24개, 원본과 별도 썸네일
- 프롬프트, 생성일, 원본 열기, 다운로드, 삭제, 저장 용량 표시

### 2.8 프로젝트와 기획서

- 빈 프로젝트 또는 Markdown 기획서로 시작
- 프로젝트 이름, 대표 썸네일, 컷 수, 수정일, 삭제
- Hook, Relate, Twist, Punchline, CTA 및 컷별 대사·텍스트·장면 지시
- 기획서 분석 후 이미지 일괄 생성
- 고급 플랫폼 화면에는 PDF, 이미지, DOCX, ZIP, TXT, 블로그 자료 가져오기, 캐릭터 별칭·역할, 스타일 참조, 의료광고 준수 옵션이 노출됨
- 목록의 컷 수, 편집 문서 페이지 수, 영상용 사용 가능 컷 수가 서로 다르게 보이는 데이터 정합성 문제가 관찰됨

### 2.9 캔버스 편집기

- 프로젝트 아래 다중 컷/페이지, 추가·이름 변경·순서·복제·삭제·표지 지정

- 이동, 크기, 회전, 텍스트, 말풍선, 도형, 펜, 지우개, 크롭
- 레이어 순서·잠금·그룹, 정렬·분배, 가이드

- 관리자·고객·에이전시·워크스페이스·프로모션·결제 관리
- 브랜드/마케팅 분석, 캠페인, 광고, 보고서, GEO, 크롤러
- 업무·스탠드업·TIL·커리어·CRM
- CX 상담원, 지식베이스, 위젯, 운영시간, 예약, 모니터링
- Instagram, Threads, Reddit, Quora, 블로그 예약 게시

이 모듈들은 Toon 제작 기능의 동등성 기준과 분리한다. 사용자의 현재 목표는 개인 창작 서비스이며 Plurank 전체 SaaS 복제는 제품 범위 결정이 먼저다.

### 2.14 제스처와 통합 스튜디오의 경계

두 화면 모두 캐릭터와 프롬프트를 받아 이미지를 만들기 때문에 겉보기에는 비슷하지만 결과의 수명과 편집 문맥이 다르다.

- **제스처 생성**은 프로젝트 밖에서 반복 사용할 캐릭터 포즈 자산을 만든다. 1인/2인, 캐릭터·스타일 참조, 배경 여부, 카메라 앵글, 품질과 독립 히스토리가 핵심이다.
- **통합 스튜디오**는 프로젝트와 현재 컷을 먼저 선택한 뒤 캐릭터·제스처·배경 자산을 조합하고 레이아웃·텍스트·말풍선·레이어를 편집해 PNG/ZIP/영상으로 내보낸다.
- 레퍼런스에서도 제스처 결과는 자산이고 스튜디오는 자산 소비자다. 둘을 하나의 화면으로 완전히 합치면 재사용 자산 생성과 현재 컷 수정이 섞여 오히려 작업 대상을 잃기 쉽다.
- TO-BE는 메뉴상 제스처를 `캐릭터` 아래에 두되, 생성 결과를 공통 `GenerationJob -> GenerationArtifact -> 작업 보관함`에 저장하고 스튜디오 자산 패널이 그대로 소비하는 방식이다.
- 스튜디오 내부 `컷 포즈`는 별도 제스처 도구의 복제가 아니라 현재 컷에 즉시 적용하는 빠른 작업으로 유지한다.

배경과 장면도 같은 원칙을 쓴다. 배경은 인물이 없는 재사용 공간 자산이고 장면은 캐릭터·행동·배경을 결합한 최종 이미지다. 한 작업공간의 두 탭으로 묶되 저장한 배경이 장면 선택 목록과 스튜디오 자산에 즉시 나타나야 실제 통합으로 판정한다.

## 3. AutoCartoon AS-IS

### 3.1 운영 구조

```text
Browser
  -> Cloud Run / Next.js 15 Route Handlers
  -> Cloud SQL PostgreSQL / Prisma
  -> GenerationJob + GenerationArtifact + CreditLedger
  -> Cloud Tasks
  -> Vertex AI Gemini / Veo + Google Cloud TTS
  -> private GCS + owner-aware media gateway
```

- 운영: `https://wonybananabot-272254743773.asia-northeast3.run.app`
- 로그인: 카카오·Google 신규 가입, 기존 이메일 회원용 레거시 로그인·12자 임시 비밀번호·강제 변경
- 세션: iron-session HttpOnly 쿠키와 DB 기기 세션 최대 2대
- 데이터: 34개 Prisma 모델, 사용자 소유권 기반 개인화
- 생성: Cloud Tasks 비동기 작업, 단계·진행률, 재접속 복구, 재시도, 작업·운영 통합 알림, 자동 환불
- 저장: GCS 원본과 WebP 썸네일, 미디어 게이트웨이 소유권 검사

### 3.2 실제 사용자 기능

- 중복 바로가기를 제거한 제작 대시보드, 온보딩, 통계, 최근 작업·프로젝트
- 구조화 캐릭터 생성과 1K/2K 품질, 결과 재시도·저장
- 캐릭터 방향별 이미지, 대표/기본 캐릭터, 그룹·마켓, 최대 3개 Chirp 3 HD 보이스
- `캐릭터` 안의 1인·2인 제스처 자산 생성과 최대 4명 일반 장면, 방향·앵글·다중 참조
- 저밀도 배경 빠른 생성과 3단계 정리·스타일·앵글 워크플로
- 배경 영속 히스토리에서 과거 결과를 새 워크플로 입력으로 재사용하고 저장 즉시 장면 배경으로 선택
- 통합 보관함의 타입 필터, 페이지 이동, 썸네일, 저장 용량, 삭제
- 프로젝트·컷·표지·순서·복제·자산·대사·영상 플랜
- Markdown, TXT, CSV, HTML, PDF, DOCX, ZIP, 이미지 OCR, 공개 URL 기획 자료 추출·저장·AI 프로젝트 변환
- 6개 비율과 7개 레이아웃, 다중 페이지, 인라인 이름·표지·순서·복제·삭제
- 레이어·잠금·그룹·클리핑·가이드·정렬·분배·직접 크기·회전·뒤집기·불투명도·필터
- 33개 폰트, 부분 문자열 서식, 커스텀 말풍선, 도형, 5종 브러시, 효과음 10종
- OCR, 배경 제거, AI 자동·사각형·자유 마스크 다시 그리기와 서버 픽셀 보존

### 4.1 2026-07-18 기능 재감사 결론

레퍼런스에 있고 AutoCartoon이 실제로 아직 부족한 기능은 다음과 같다.

1. **카카오 알림톡 완료 통지**: SOLAPI HMAC 어댑터는 준비했지만 사용자 휴대전화번호·수신 설정과 작업 완료 이벤트가 없다. 비즈채널, 발신프로필, 템플릿 승인도 필요하다.
2. **휴대폰 본인확인**: 가입·복구는 카카오·Google·이메일 보안 흐름으로 동작하지만 레퍼런스의 휴대폰 실명 확인과 같지는 않다.
3. **운영 결제**: 크레딧 상품, 원장, 카카오페이 호출 코드는 있으나 가맹점 승인 전 비활성인 것이 정상이다.
4. **SNS 직접 게시**: Instagram 코드는 있으나 Meta 검수 전 메뉴가 비활성이다.

다국어는 국내 전용 제품 결정에 따라 기능 공백에서 제외했다. 법적 문서 3종은 화면과 공통 링크까지 구현했으며 현재는 사업자 정보와 법률 검토를 기다리는 운영 전 초안이다.

이번 재감사에서 부족하다고 오인하기 쉬우나 이미 동등 이상으로 확인한 영역:

- 캐릭터 구조화 생성, 4방향 관리, 보이스 3개와 미리듣기
- 1인·2인 제스처, 다중 참조, 10종 이상 카메라 앵글, 영속 히스토리
- 배경 원본·스타일·텍스트 입력, 저밀도 생성, 정리·스타일·앵글 3단계, 기록 재사용
- 타입별 보관함, 썸네일, 다운로드·삭제, 프로젝트·기획서, 고급 캔버스, 30컷 영상
- 광장 최신·인기·댓글·신고에 더해 글/댓글 좋아요와 관리자 고정

사용자 API 키, 공급자 선택, 구독 상품은 의도적으로 다르다. AutoCartoon은 사용자가 키를 넣지 않고 플랫폼 Vertex AI와 크레딧을 쓰는 제품 방향이므로 레퍼런스 BYOK 복제는 기능 공백으로 계산하지 않는다.

## 5. 이번 TO-BE 즉시 반영

1. 전용 제작 홈을 추가해 자산 수, 크레딧, 온보딩, 최근 작업·프로젝트를 한 화면에서 복구한다.
2. PDF/DOCX/ZIP/Markdown/TXT/CSV/HTML, 이미지 OCR, 공개 URL 자료를 크기 제한과 SSRF 방어 뒤 AI 기획 입력으로 연결한다.
3. 카카오 이메일 권한이 없는 사용자가 만든 내부 계정을 기존 이메일 계정에 명시적으로 연결한다. 콘텐츠·결제 데이터가 있는 계정은 자동 병합하지 않는다.
4. DB 지식이 비어도 챗봇이 현재 기능, 크레딧, 결제 상태를 정확히 답하도록 내장 서비스 지식을 제공한다.
5. 배경 히스토리 결과를 스타일 변환·앵글 생성의 입력으로 즉시 재사용한다.
6. 게시판 좋아요 실패 시 낙관적 UI를 되돌려 DB와 화면 수치가 달라지지 않게 한다.
7. Google Storage 전이 의존성의 보안 경고를 수정된 `uuid`로 고정하고 전체 호환성을 검증한다.
8. 고급 캔버스를 레퍼런스 전 도구와 재대조하고 직접 변환, 페이지 관리, 커스텀 말풍선, 영역 AI, 버전 복원과 변경 레이어 저장을 완성한다.
9. 운영 공지의 관리자 작성·게시·고정·만료·삭제, 홈 표시, 작업 알림과의 통합, 사용자별 읽음 상태를 추가한다.
10. 국내 서비스용 이용약관, 개인정보처리방침, 크레딧·환불정책 초안과 공통 링크를 추가한다.
11. SOLAPI 어댑터의 인증을 공식 HMAC-SHA256 규격으로 수정하고 알림톡 도입 절차를 문서화한다.
12. 상단 정보 구조를 `캐릭터 | 배경/장면 | 통합 스튜디오 | 숏폼 | 내 작업`으로 줄이고 게시판·WonyBot·계정 아이콘을 보조 동선으로 분리한다.
13. 제스처와 배경을 공통 생성 작업·산출물·보관함·스튜디오 자산 파이프라인에 연결하고 비활성 패널의 API 폴링을 중단한다.
14. 전체 UI를 톤다운된 밝은 파스텔 표면과 낮은 반경의 현대적 버튼 체계로 변경한다.

## 6. 남은 결정과 구현 경계

코드만으로 임의 확정하면 안 되는 항목:

1. **법적 문서 최종화**: 대표자, 사업자번호, 주소, 개인정보 담당자, 위탁·국외 이전 계약, 환불 산식과 법률 검토. 화면 초안은 구현 완료.
2. **휴대폰 본인확인**: PASS/카카오/통신사 공급자, 가입 차단 수준, 비용과 개인정보 보관 정책.
3. **카카오 알림톡**: 비즈채널, 발신 프로필, 수신 전화번호 처리, 템플릿 심사와 건당 비용. [실행 문서](./kakao-alimtalk-setup.md) 참고.
4. **결제**: 현재 미작동이 정상. 카카오페이 가맹점 심사, 운영 CID, 정책·세금·정산 기준 확정 후 활성화.
5. **SNS 게시**: Meta 앱 검수, 장기 토큰, 예약 게시 채널과 운영 책임.
6. **음성 복제**: 공급자, 화자 동의, 상업 이용·초상/성문권 계약.
7. **Plurank 확장군**: CRM·CX·마케팅·광고·업무 모듈을 개인 창작 제품에 포함할지 별도 결정.

코드상 다음 개선 후보:

- 매우 큰 숏폼 프로젝트를 위한 서버 렌더 워커
- 보관함과 태그·즐겨찾기·프로젝트 자산의 단일 검색 인덱스
- 캔버스 버전 보존량과 AI 영역 수정 비용의 요금제별 정책
