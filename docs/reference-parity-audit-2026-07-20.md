# ToonAgent 레퍼런스 vs AutoCartoon 기능 패리티 감사 — 2026-07-20

## 0. 조사 방법과 범위

- 레퍼런스: `https://app.toonagent.co.kr` (권혜영 계정, 프론트엔드만 접근 — 계정주 허락, 저장/생성/삭제 등 데이터·비용 변경 행위 없음, 만든 임시 객체는 Undo로 원복)
- 우리 구현: 로컬 소스 전수 분석(`CanvasEditor.tsx` 5300줄 등) + 로컬 DEV 브라우저 실조작
- 방법: 레퍼런스 캔버스 에디터의 8종 도구 레일과 모든 옵션 패널·모달을 접근성 트리/DOM으로 1개씩 열어 실제 옵션·enum·기본값을 기록하고, 우리 소스 인벤토리와 항목 단위로 대조
- 핵심 결론 먼저: **기능 커버리지는 사실상 완전 이식(대부분 오버스펙)이며, 캔버스는 메뉴 1:1로 거의 완벽 카피됨.** 단, 뎁스 레벨에서 **실제 사용자 접근이 막힌 갭 2건**과 **아키텍처 의도 divergence 3건**을 확인했다.

## 3. 실제 뎁스 갭 (사용자 접근 관점) — 조치 필요

레퍼런스는 되는데 **우리는 코드는 있으나 사용자가 실제로 쓸 수 없는** 항목. 이게 "뎁스 내부까지 카피됐는가"의 핵심 답이다.

### A. 텍스트 부분 문자열 서식 — **기준선만 되고 색/굵기/기울임/밑줄은 접근 불가**

- **레퍼런스**: 텍스트 옵션에 "적용할 텍스트를 먼저 선택해주세요" 안내와 함께 **부분서식 버튼 6개를 라이브 노출**. 텍스트 일부만 색·굵기·기울임·밑줄을 다르게 줄 수 있음(웹툰 강조 표현의 핵심).
- **우리**: 엔진(`textRuns`, 최대 120런)은 완성돼 있으나, 라이브 텍스트 패널(`CanvasEditor.tsx:4130-4174`)에서 색/굵기/기울임/밑줄은 전부 `updateTextTool()`=**전체 적용**만 호출. 부분 적용(`applySelectedTextStyle`)은 **기준선 ↑/↓ 2개에만** 배선됨. 색/굵기/기울임/밑줄의 부분 적용 버튼은 `legacyToolbar`(`CanvasEditor.tsx:4447`, `aria-hidden="true"`, CSS `display:none`, `toolbarCollapsed=false`로 항상 숨김)에만 존재해 **사용자 도달 불가**.
- **주의**: 우리 문서 [advanced-canvas-parity.md](advanced-canvas-parity.md) 82행은 "부분 문자열 색·굵기·기울임·밑줄·기준선"을 우위로 기재 — **실제 라이브 UI와 불일치(문서 과장 또는 UI 회귀)**.
- **조치**: `legacyToolbar`의 부분서식 배선(색/굵기/기울임/밑줄)을 라이브 텍스트 패널로 이식. 엔진이 이미 있으므로 UI 배선만으로 해결 가능(경량). 우선순위 **높음** — 사용자가 명시적으로 강조한 뎁스 영역.

### B. 커스텀 말풍선 "라이브러리 저장" — **write-only(불러오기 없음)**

- **레퍼런스**: 말풍선 생성기에 "이 말풍선을 내 라이브러리에 저장" 버튼(재사용 목적).
- **우리**: `CanvasEditor.tsx:4259`가 `localStorage.setItem("wony-canvas-custom-bubble", ...)`로 **쓰기만** 하고, `getItem` 호출이 코드 전체에 없음(grep 확인). 저장해도 다시 불러오는 UI/로직이 없어 **사실상 무동작 버튼**.
- **조치**: (1) 저장 슬롯 불러오기 UI 추가로 기능 완성, 또는 (2) 오해를 주는 버튼 제거. 우선순위 **중간**.

### C. 코드 존재·UI 미노출 (기능 손실 아님, 참고)

- 말풍선 `angry`/`needle`, 도형 `star`가 렌더러엔 있으나 라이브 그리드에 버튼 없음. 레퍼런스도 기본 말풍선 3종·도형 5종이라 **패리티엔 영향 없음**(오히려 우리 여분 자산). 원하면 노출로 오버스펙 가능.

## 4. 아키텍처 의도 divergence (이식 대상 아님 — 확인 후 결정 사항)

우리 프로젝트 메모리·핸드오프 문서에 **의도적 divergence로 이미 기록된 항목**. 레퍼런스를 그대로 베끼면 안 되는 부분이니 의뢰자 결정이 필요.

| 항목 | 레퍼런스 | 우리 | 성격 |
| --- | --- | --- | --- |
| AI 비용 모델 | **BYOK**(사용자 Gemini/OpenAI 키, 플랫폼은 SW 이용료만) | **플랫폼 소유 Vertex**(크레딧 과금, 키 비노출) | 의도적 — 진입장벽↓·보안↑ 대신 원가 부담 구조 상이 |
| 과금 구조 | 구독제(기간권/자동갱신) + 부분 크레딧 | 크레딧 단일 | 의도적 — 정책 미확정 |
| 이미지 모델 선택 | Gemini/Gemini Pro/ChatGPT 사용자 선택 | Vertex 고정 | BYOK 부재의 결과 |
| 다국어 | 한/영/일 | 한국어 전용 | 의도적(국내 전용) |
| 휴대폰 본인확인 | 있음 | 미구현 | 외부 결정 대기 |

> 판단 포인트: 의뢰자가 "레퍼런스 기능을 상당량 이식"을 원한다면, **BYOK + 이미지 모델 선택 + 구독제**는 캔버스와 달리 제품 방향을 바꾸는 결정이라 별도 합의가 필요하다. 캔버스/제작 파이프라인은 이미 이식 완료 상태다.

## 5. 최종 판정

1. **기능 커버리지**: 레퍼런스의 제작·편집·공유 파이프라인 전 기능이 우리에 존재하며, 규격·필터·버전관리·OCR·AI 마스크 보존 등 다수 항목은 오버스펙. **커버리지 목표 달성.**
2. **캔버스 1:1 카피**: 도구 8종·팔레트 hex·효과음 10종·페이지 배경·워터마크·캡션·레이아웃·말풍선 생성기·AI 다시그리기까지 **메뉴 단위로 거의 완벽 일치**(상당수 우위).
3. **뎁스 갭 2건**(§3): ① 텍스트 **부분 문자열 색/굵기/기울임/밑줄**이 숨겨진 legacy 툴바에 묶여 접근 불가(엔진은 완성 — UI 이식만 필요), ② 커스텀 말풍선 저장이 write-only. 둘 다 레퍼런스에는 라이브로 존재하므로 **실제 이식 완결을 위해 보완 권장.**
4. **아키텍처 divergence 3건**(§4): BYOK·구독제·다국어는 의도적 차이 — 이식 여부는 제품 결정.
