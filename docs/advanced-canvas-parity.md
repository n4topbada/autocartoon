# 고급 캔버스 역설계와 동등성 검증

최종 갱신: 2026-07-23 KST

- 레퍼런스: `https://app.toonagent.co.kr/instatoon/projects/:id`
- 구현: `/studio`의 `CanvasEditor`
- 목표: 레퍼런스와 같은 편집 목적을 모두 달성하고 정밀 편집, 복구, 저장 비용에서는 더 나은 사용성을 제공한다.
- 접근 비밀은 Git에서 제외한 `docs/access-credentials.private.md`에서만 관리한다.

## 1. 조사 방법

인증된 레퍼런스의 실제 프로젝트에서 각 도구와 접이식 패널을 열어 접근성 트리, 입력값, 상태 변화와 사용자 흐름을 확인했다. 저장·AI 생성처럼 데이터나 비용을 바꾸는 최종 제출은 테스트 프로젝트에 한정했고, 브랜드 자산·문구·코드는 복제하지 않았다.

확인한 표면:

- 프로젝트 헤더, 저장 상태, 캔버스 규격, 레이아웃, 업로드와 내보내기
- 선택·브러시·지우개·텍스트·말풍선·도형·스포이트·OCR
- 정렬·분배·잠금·그룹, 이미지 객체, 캐릭터·제스처·배경 자산
- 워터마크·캡션·효과음, 가이드·투명 영역·넘침, 크롭·누끼
- 직접 그리기, 페이지 배경, 레이어, 다중 페이지
- AI 자동 영역 또는 직접 마스크 기반 컷 다시 그리기

## 2. 레퍼런스 동작

### 문서와 페이지

- 규격 4종: 1080x1080, 1080x1350, 960x1280, 800x1100

## 4. 구현 구조

```text
CanvasEditor
  -> Canvas 2D 공통 합성기
  -> Layer[] + SpeechBubble[] + PageBackground
  -> 변경 레이어 업로드 티켓
  -> POST /api/images/save
  -> ProjectCut.canvas + CanvasVersion

일괄 프리셋
  -> 현재 컷 브라우저 렌더·즉시 저장
  -> POST /api/studio/projects/:id/canvas-presets
  -> 전체/범위 ProjectCut.canvas 갱신 + 변경 전 CanvasVersion
  -> 전체 ZIP에서 각 컷 최신 구조를 다시 렌더

누끼
  -> 코너 색상: 4개 코너 색 표본(명백한 예외 1개 제외)
     -> 허용 오차 내 연결 영역만 flood fill -> 브라우저에서 즉시 투명화, 무료
  -> AI 정밀: GET /api/studio/remove-background 연결 상태
     -> 13크레딧 원장 예약 -> Nano Banana 2 + 크로마키 투명화
     -> 원본 피사체 경계로 결과 위치·크기 보정 -> 성공 PNG / 실패 자동 환불

AI 영역 수정
  -> 전체 / AI 자동 / 사각형 / 자유 마스크
  -> POST /api/generate
  -> GenerationJob + Cloud Tasks
  -> Vertex Gemini 이미지 수정
  -> Sharp: generated DEST-IN mask OVER original
  -> ProjectCut 갱신 + 이전/결과 CanvasVersion
```

주요 파일:

- `src/components/CanvasEditor.tsx`: 편집 상태, 도구, 직접 조작, 저장·복원 UI
- `src/components/CanvasEditor.module.css`: 고정 패널, 반응형 도구·페이지·이력 UI
- `src/lib/bubble-draw.ts`: 말풍선·텍스트·도형 공통 렌더러와 회전 좌표
- `src/lib/canvas-presets.ts`: 워터마크·캡션 구조 정규화와 일괄 변경
- `src/lib/corner-cutout.ts`: 코너 예외 판별, 유사색 연결 영역 누끼, AI 위치 보정용 전경 경계 계산
- `src/app/api/studio/projects/[id]/canvas-presets/route.ts`: 소유권 기반 전체·범위 프리셋 저장
- `src/app/api/studio/remove-background/route.ts`: Nano Banana 누끼와 투명 PNG 변환, 크레딧 차감·환불
- `src/lib/generation-service.ts`: 자동 영역 감지, 생성 결과 마스크 합성
- `src/app/api/images/save/route.ts`: 컷 저장과 버전 생성

- `prisma/migrations/20260717190000_add_canvas_versions`: 버전 테이블

## 5. 저장과 복구 정책

- 캔버스 JSON 버전은 `3`이며 구버전 `1`, `2`도 읽는다. 페이지 배경은 특수 레이어가 아니라 루트 `pageBackground`로 저장한다.
- 픽셀 자체가 바뀐 레이어만 새 PNG를 올린다. 이동·크기·회전·필터·텍스트 변경은 기존 픽셀 URL을 재사용한다.
- 1.6초 유휴 자동저장은 변경된 구조와 픽셀 레이어만 저장하며 버전을 늘리지 않는다. 수동 저장과 AI 생성은 복구 가능한 새 버전을 남긴다.
- 합성 결과 버전은 컷당 최신 60개만 유지한다.
- 컷 저장은 생성 아카이브에 `GeneratedImage`를 중복 생성하지 않는다.
- 복원 전 현재 상태를 자동 백업하고 이미지와 세부 레이어 JSON을 즉시 함께 복원한다.
- AI 결과는 이전 캔버스를 자동 백업한 뒤 새 원본으로 반영한다.
- 버전 정리 시 직접 합성 이미지와 썸네일은 다른 DB 참조가 없을 때만 삭제한다.

## 6. 검증 기록

- 변경 전 첫 저장: 레이어 PNG + 합성 PNG 2회 업로드
- 변경 없는 재저장: 합성 PNG 1회 업로드
- 픽셀 편집 뒤 저장: 변경 레이어 PNG + 합성 PNG 2회 업로드
- 모서리 직접 리사이즈: 540x675 -> 378x472, 미저장 상태 표시
- 레이어 직접 회전: 17도 반영
- 텍스트 직접 회전: 62도 반영
- 페이지 이름 인라인 변경·복원과 표지 지정 API 성공
- 버전 복원 시 레이어 이름·JSON 복원과 `복원 전 자동 백업` 생성 확인
- 마스크 밖 픽셀 동일성 및 회전 좌표·히트 테스트 자동화
- 워터마크 중복 방지·삭제 보존, 캡션 선택 적용, 구버전 기본값 복구 자동화
- 텍스트·도형·말풍선 드래그 생성, 레이어 드래그 순서, 선택 해제 후 컷 방향키 이동 구현
- 레퍼런스 37개 폰트와 8색 팔레트 실 DOM 값 대조
- 레퍼런스 도구 8종을 상태별로 다시 열어 생성 전 기본값, 선택 후 속성, 단계형 적용 흐름을 재대조
- 로컬 브라우저 E2E: 8개 도구 패널, 텍스트 72px 생성·재선택, 화살표 생성, 투명 지우개 적용, 직접 그리기 5종·6색 확인
- 로컬 브라우저 E2E: 커스텀 말풍선 4형태와 W/H·흔들림·선·채움·꼬리·저장/PNG/추가/닫기 확인
- 로컬 브라우저 E2E: 레이아웃 7종, 배경 4방식, 효과음 10종, AI 영역 4방식, 워터마크·캡션 설정 확인

## 7. 남은 운영 선택

캔버스 코드와 UI의 확인된 필수 기능 공백은 없다. 다음 항목은 외부 연동 또는 운영 정책이다.

- AI 정밀 누끼는 기존 Vertex AI/Nano Banana 인증을 재사용한다. 모델 실패나 피사체 분리 실패는 원장 트랜잭션에서 자동 환불한다. 단색·스튜디오 촬영 배경은 무료 코너 색상 방식을 먼저 쓰는 편이 빠르고 원본 픽셀 보존에도 유리하다.
- 교보손글씨 2025 외부 폰트 URL은 제공처 배포 경로가 바뀔 수 있어, 운영 전 자체 GCS 호스팅 여부를 결정한다.
- 버전 60개 보존 기간을 사용자 요금제별로 다르게 할지
- 대형 프로젝트의 ZIP·영상 내보내기를 서버 워커로 옮길지
- 폰트를 Google Fonts 대신 자체 호스팅해 폐쇄망·초기 로딩을 고정할지
- AI 영역 수정 비용을 전체 수정과 다르게 책정할지
